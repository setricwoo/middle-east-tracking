# 切换到每小时定时任务说明

## 一、停止现有的20分钟定时任务

### 方法1：如果正在运行命令行窗口
直接按 `Ctrl+C` 停止

### 方法2：通过任务管理器停止
1. 按 `Ctrl+Shift+Esc` 打开任务管理器
2. 查找 Python 进程
3. 结束所有与 scheduler.py 相关的进程

### 方法3：通过任务计划程序停止（如果是Windows计划任务）
1. 按 `Win+R`，输入 `taskschd.msc` 打开任务计划程序
2. 查找名为 `MiddleEastTracking` 或类似的任务
3. 右键删除或禁用该任务

## 二、启动新的每小时定时任务

### 方法1：直接运行（推荐测试时使用）
双击运行 `start_hourly_scheduler.bat`

这会打开一个命令行窗口，每小时自动执行：
1. 更新冲突每日简报
2. 更新实时新闻
3. 推送到GitHub

**注意**：关闭窗口即停止任务

### 方法2：创建Windows计划任务（推荐长期使用）
1. 右键点击 `setup_hourly_task.ps1`，选择"使用 PowerShell 运行"
2. 或者在 PowerShell 中执行：
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process -Force
   .\setup_hourly_task.ps1
   ```
3. 任务将每小时自动运行，无需保持窗口打开

## 三、每小时任务执行内容

新的每小时调度器 (`hourly_scheduler.py`) 会按顺序执行：

### 1. 更新冲突每日简报
- 运行 `update_briefing_grok.py`
- 使用Grok AI生成最新简报内容
- 更新 `briefing.html`

### 2. 更新实时新闻
- 运行 `scrape_cls_final.py`
- 爬取财联社最新新闻
- 增量更新 `news.html`（保留旧新闻，新新闻置顶）

### 3. 推送到GitHub
- 自动提交所有更改
- 推送到远程仓库

## 四、查看执行日志

所有执行记录保存在 `scheduler_log.txt` 中

## 五、手动执行单次更新

如果需要立即执行一次更新，可以运行：

```bash
# 更新简报
python update_briefing_grok.py

# 更新新闻
python scrape_cls_final.py

# 推送到GitHub
git add -A
git commit -m "手动更新"
git push origin main
```

## 六、常见问题

### Q: 推送GitHub失败怎么办？
A: 脚本会自动重试，如果仍然失败，请检查：
- 网络连接
- GitHub仓库访问权限

### Q: 如何修改执行频率？
A: 编辑 `hourly_scheduler.py`，修改以下行：
```python
schedule.every(1).hours.do(run_hourly_update)  # 改为需要的频率
```

可选频率：
- `schedule.every(30).minutes.do(...)` - 每30分钟
- `schedule.every(2).hours.do(...)` - 每2小时
- `schedule.every().day.at("10:00").do(...)` - 每天10点

### Q: 只想更新部分内容？
A: 可以注释掉不需要的步骤：
```python
def run_hourly_update():
    # briefing_ok = run_briefing_update()  # 注释掉这行就不更新简报
    news_ok = run_news_update()
    git_ok = run_git_push()
```
